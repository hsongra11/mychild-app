# Q12-Q18 Threshold Worksheet (Ruleset v0.1 Hypothesis)

Purpose: define and review threshold hypotheses for the new MVP questions (Q12-Q18) before clinical activation.

Status policy:

1. `DRAFT`: unvalidated hypothesis.
2. `PHASE_B_READY`: ready for retrospective simulation.
3. `PHASE_C_READY`: ready for prospective pilot.
4. `CLINICAL_APPROVED`: eligible for production use.

Gating rules:

1. No Q12-Q18 question may drive `warning` or `flag` without clinical reviewer sign-off.
2. All cutoffs must include a written rationale and source.
3. Use corrected-age rule for preterm infants where applicable.
4. Keep `Ruleset v0.1 (hypothesis)` tag until Phase B and Phase C evidence is documented.

## Fill Instructions

1. Set `Hypothesis Source` to the framework used (for example CDC milestone range, ASQ-style interval, advisor consensus).
2. Enter candidate `Precaution/Warning/Flag` months.
3. Add caregiver-facing observability prompt and concise clinical rationale.
4. Record simulation and pilot outcomes after each phase.
5. Complete sign-off fields before moving to `CLINICAL_APPROVED`.

## Question List

1. Q12: Responds to own name (N), age 6-12 months
2. Q13: Pincer grasp emerges (P), age 8-14 months
3. Q14: Points to show interest (N,S), age 10-16 months
4. Q15: First meaningful words (S), age 10-18 months
5. Q16: Follows simple instructions (N,S), age 12-20 months
6. Q17: Pretend play emerges (N), age 16-24 months
7. Q18: Two-word combinations (S), age 18-24 months

## Review Checklist

1. Is this milestone observable by a non-clinician caregiver?
2. Is the threshold age internally consistent with adjacent milestones?
3. Could this question generate high concern from a single weak signal?
4. Is the notification copy non-alarming and action-oriented?
5. Are there confounders (language environment, hearing issues, prematurity) requiring caveats?
6. Does the rule meet minimum-evidence gating for domain escalation?
